Coding Style Guide

Commenting

Use of comments is highly encouraged to increase readability of code. Every class method should have a comment before declaration that explains the purpose of the method and its return values, if any. Within each method, comments should be clear and concise, so as not to over-complicate the method. Only include a comment if the particular piece of code you are referencing is challenging to read. Always place a comment on its own line and before the piece of code it references.

The start of every .cpp or .h file should contain a comment detailing the name of the file and the author. This comment is of the following format:

//
// MyFile.h
// Created by [Your Name]
//

Printing to the Terminal

Never use calls to the printf() function. Instead, rely on the standard library for printing to the terminal. In any given .cpp file, include the following code before method declarations:

#include <stdlib.h>

using std::cout;
using std::endl;

Then, within the method, you may print to the terminal using the following format:

cout << “[your message here]” << endl;

Variables may be printed using the standard library as well, such as:

cout << “This is my variable: “ << myVariable << endl;

Brackets

The most important rule for bracket usage is that the left bracket should remain on the same line as the statement and above the code it contains. This is utilizing the K&R bracing style. Several examples are given below.

Function/Method Declarations

YourApplication::someFunction(int aNumber) {
    // Code here
}

If-Else Statements

For if-else statements, subsequent conditions should appear on the same line as the right bracket of the previous condition.

if (condition) {
    // Code here
} else if (condition) {
    // Code here<
} else {<
    // Code here
}

For-Loops

Unlike if/else statements, each for-loop should start on its own line and should not follow a right bracket.

for (i;i<j;i++) {
    // Code here
}

for (k;k<l;k++) {<
    // Code here
}

Indentation

Indentation within methods should follow a Russian-doll nesting model. One indent (tab key) should be used for every piece of code that is nested within a for-loop, if-else statement, function/method declaration, or while-loop.

YourApplication::SomeMethod(int aNumber) {
    if (condition) {
        for (i;i<j;j++) {
            while (condition) {
                // Code here
            }
        }
    }
}

Declaring a Method/Function

Method/Function names should not be capitalized, but all subsequent words within the method name should be capitalized.

YourApplication::yourFunction() {

}

Any method that gets a member variable should be named:

get[Variable name]();

Similarly, any method that sets a member variable should be named:

set[Variable name]();

All other methods are to be named at the discretion of the programmer. However, it is recommended that function names be concise and convey the purpose of the function.

General Coding Rules

  • Delete all debugging calls to printf() or cout/endl.
  • Declare variables at the beginning of the function/method.
  • Use new and delete instead of malloc and free.
  • Class declarations should be capitalized and use CamelCase.
  • Pointers should be declared with the * butting up next to the type: int* myInt.
